# Лабораторная работа №3

## Описание

Лаба представляет собой переделанную лабораторку №1, но пайпы заменены на SharedMemory

### Выводы

Что я узнал из лабораторной работы:
- В линуксе минимальные блоки памяти, которыми управляет ядро, называется страницами памяти, обычно равные 4кб. Физическая и виртуальная память делится на страницы
- mmap позволяет создавать отображения файлов или устройств в виртуальное адресное пространство вызывающего процесса
- - Аргумент addr это предпочтительный адрес отображения, он может быть null, что является более "portable" способом
- - Минимальный виртуальный адрес с которого система может начать создавать маппинги в виртуальном адресном пространстве процесса находится в `/proc/sys/vm/mmap_min_addr`
- shm_open создаёт именнованный объект разделяемой памяти, по факту, создаёт файл в /dev/shm/

SharedMemory это мощный инструмент для коммуникации между процесссами, но в отличии от пайпов он не имеет интерфейса очереди (а произвольный доступ) и в некоторых задачах из-за этого приходится придумывать способ синхронизации сообщений
